# SonaKeep App Store Submission Packet

This directory contains the repository-side submission packet for App Store preparation.

## Files
- `metadata.md` — app name, subtitle, keywords, description, promotional text, and release notes
- `screenshots-plan.md` — screenshot shot list, captions, and demo-data guidance
- `review-notes.md` — App Review notes draft and privacy-label draft

## What still needs to happen manually
- Capture actual screenshots from a polished preview/production build
- Publish the support/privacy pages and replace placeholder URLs in metadata
- Enter the final values into App Store Connect
- Verify the final category, age rating, and privacy answers in App Store Connect UI

## Recommended sequence
1. Merge this packet
2. Publish the `site/` pages and get live HTTPS URLs
3. Capture screenshots from the latest release-candidate build
4. Paste the copy from `metadata.md` into App Store Connect
5. Paste reviewer notes from `review-notes.md`
6. Submit TestFlight / App Review build
