# SonaKeep Screenshot Plan

Capture screenshots from a polished preview or production build on the required iPhone sizes.

## General guidance
- Use realistic but clean demo data
- Keep Privacy Mode off for screenshots unless a privacy-specific screenshot is intentional
- Avoid placeholder or obviously fake item names
- Ensure rates source state is coherent in the screenshot session
- Prefer consistent currency and weight unit across the set

## Suggested screenshot set

### 1. Dashboard overview
**Screen:** Home dashboard with populated portfolio

**What to show:**
- total portfolio value
- total weight
- item count
- recent items section

**Caption idea:**
Track your entire gold portfolio at a glance.

### 2. Holdings list
**Screen:** Holdings with multiple items

**What to show:**
- varied item categories
- values visible
- premium/clean list state

**Caption idea:**
Organize jewelry, coins, bars, and more in one place.

### 3. Item detail
**Screen:** Item detail for a representative item

**What to show:**
- current value
- weight, purity, purchase price, date, location
- gain/loss section if available

**Caption idea:**
Keep the details that matter for every item.

### 4. Rates screen
**Screen:** Rates view with source banner visible

**What to show:**
- 24K spot price
- purity-specific rates
- source label such as Live data or Cached data

**Caption idea:**
Monitor current valuation with clear rate-source labeling.

### 5. Privacy / settings
**Screen:** Settings with Privacy Mode and biometric controls visible

**What to show:**
- Privacy Mode row
- biometric lock row
- local storage explanation

**Caption idea:**
Built to feel discreet, private, and calm.

## Optional sixth screenshot

### Empty-state onboarding
**Screen:** Home empty state

**What to show:**
- trust cards
- Add First Item CTA

**Caption idea:**
Start tracking your gold collection in minutes.

## Capture checklist
- [ ] Use final app icon and branding
- [ ] No debug banners or dev overlays
- [ ] No placeholder text
- [ ] No broken timestamps or fallback states unless intentionally shown
- [ ] No embarrassing demo notes or locations
- [ ] Align captions with actual visible UI
